# Преобразователь регулярной грамматики в конечный автомат

## Описание проекта

Программа принимает на вход текст регулярной грамматики (праволинейной или леволинейной) и строит соответствующий конечный автомат. Если полученный автомат оказывается недетерминированным, выполняется его детерминизация. Результат выводится в виде списка рёбер графа в текстовом формате.

## Планируемая логика работы

### Шаг 1: Ввод грамматики
**Функция:** main (модуль Main.hs)
- Выводит приглашение пользователю
- Читает строку через getLine

### Шаг 2: Разбор
**Функция:** parseGrammar (модуль Grammar.hs)
- Принимает строку вида "H=aN|bN;N=cN|d"
- Разбивает по ; на отдельные правила
- Каждое правило разбивает по = на левую и правую части
- Правую часть разбивает по | на альтернативы
- Каждую альтернативу преобразует в тип Alternative
- Возвращает Grammar

### Шаг 3: Определение типа
**Функция:** getGrammarType (модуль Grammar.hs)
- Смотрит на первый ключ в Grammar (начальный нетерминал)
- Если H, то возвращает `RightLinear`
- Если S, то возвращает `LeftLinear`
- Иначе — ошибка

### Шаг 4: Построение НКА
**Функция:** grammarToNFA (модуль Convert.hs)

**Для праволинейной грамматики:**
- Добавляется финальное состояние #
- Правило A = a -> переход A -a-> #
- Правило A = aB -> переход A -a-> B
- Начальное состояние — H

**Для леволинейной грамматики:**
- Добавляется начальное состояние @
- Правило A = a -> переход @ -a-> A
- Правило A = Ba -> переход B -a-> A
- Финальное состояние — S

### Шаг 5: Проверка детерминированности
**Функция:** isDeterministic (модуль Determinize.hs)
- Проходит по всем переходам в nfaTransitions
- Группирует переходы по (State, Symbol)
- Если для любой пары (State, Symbol) множество целевых состояний содержит больше одного элемента -> False
- Иначе -> True

### Шаг 6: Детерминизация 
**Функция:** nfaToDFA (модуль Determinize.hs)

1. Начальное состояние ДКА = {начальное_состояние_НКА}
2. Для каждого нового множества состояний:
   - Для каждого символа:
     - Переход = объединение всех переходов из каждого состояния этого множества по данному символу
     - Если полученное множество ещё не встречалось -> добавить как новое состояние ДКА
3. Допускающие состояния ДКА = множества, содержащие хотя бы одно допускающее состояние НКА

### Шаг 7: Вывод результата
**Функции:** nfaToEdges / dfaToEdges, edgesToString (модуль Automaton.hs)

**Если автомат детерминирован:**
- nfaToEdges преобразует НКА в список рёбер
- Для каждого перехода (from, sym) -> toSet создаётся ребро для каждого to

**Если автомат недетерминирован:**
- dfaToEdges преобразует ДКА в список рёбер
- Для каждого перехода (from, sym) -> to создаётся одно ребро

**Форматирование:**
- edgesToString принимает список рёбер
- Для каждого ребра (from, sym, to) формирует строку "from -> sym -> to"
- Соединяет строки через \n

**Вывод на экран:**
- main выводит результат через putStrLn
